## ADDED Requirements

### Requirement: 币种配置
系统 SHALL 从 `config.ini` 加载币种缺省参数，支持的币种至少包括 BTCUSDT、ETHUSDT、BNBUSDT。

每种币种配置包括：
| 配置项 | 说明 |
|--------|------|
| name | 币种显示名 |
| entry_price | 参考入场价（USDT） |
| upper_pct | 默认上限百分比（相对入场价） |
| lower_pct | 默认下限百分比（相对入场价） |
| grid_size | 默认网格间距（USDT） |
| quantity_per_grid | 单次网格购买数量缺省值 |
| capital | 初始资金缺省值（USDT） |

#### Scenario: 加载 BTCUSDT 配置
- **WHEN** 系统启动
- **THEN** config.ini 中包含 BTCUSDT、ETHUSDT、BNBUSDT 三个币种配置

#### Scenario: 切换币种填充缺省值
- **WHEN** 用户在表单中选择币种
- **THEN** 系统自动填充该币种的 entry_price / upper / lower / grid_size / quantity_per_grid / capital 到表单字段

### Requirement: 参数输入表单
系统 SHALL 提供一个 Web 表单，收集以下参数：

| 参数 | 类型 | 必填 | 缺省来源 | 说明 |
|------|------|------|----------|------|
| symbol | 枚举 | 是 | config.ini 首币种 | 交易对，如 BTCUSDT |
| capital | 正数 | 是 | config.ini 的 capital | 初始资金（USDT） |
| leverage | 正整数 | 是 | 1 | 仓位放大倍数，范围 1-125 |
| entry_price | 正数 | 是 | config.ini 的 entry_price | 当前标的价格，USDT |
| upper_price | 正数 | 是 | entry_price × (1 + upper_pct/100) | 须大于入场价 |
| lower_price | 正数 | 是 | entry_price × (1 - lower_pct/100) | 须小于入场价 |
| grid_size | 正数 | 是 | config.ini 的 grid_size | 相邻挂单价格间距，USDT |
| quantity_per_grid | 正数 | 是 | config.ini 的 quantity_per_grid | 单次网格购买数量（币） |

#### Scenario: 首屏渲染缺省值
- **WHEN** 用户首次访问表单页（GET /）
- **THEN** 服务端渲染时自动填充首个币种的缺省参数，所有字段非空

#### Scenario: 提交完整参数
- **WHEN** 用户填写所有必填字段并提交
- **THEN** 系统展示测算结果页面

#### Scenario: 缺少必填字段
- **WHEN** 用户未填写某个必填字段就提交
- **THEN** 系统在表单中提示该字段为必填，不提交测算

#### Scenario: 逻辑校验失败（下限 >= 入场价）
- **WHEN** 用户提交的 lower_price ≥ entry_price
- **THEN** 系统提示"网格下限必须小于入场价"

#### Scenario: 杠杆超出范围
- **WHEN** 用户提交杠杆为 0 或 >125
- **THEN** 系统提示"杠杆必须在 1-125 之间"

### Requirement: 表单数据保留
系统 SHALL 在以下场景中保留用户已提交的参数值，不重置为配置缺省值：
- 从测算结果页返回表单时（"重新测算"链接）
- 表单校验失败重新渲染时

仅当用户**首次访问**（无历史输入）时，才填充配置缺省值。

#### Scenario: 从结果页返回
- **WHEN** 用户完成测算后点击"重新测算"
- **THEN** 表单显示用户上次提交的参数，而非配置缺省值

#### Scenario: 校验失败回显
- **WHEN** 用户提交了无效参数（如下限 > 入场价）
- **THEN** 表单保留用户已填写的值，仅标出错误字段

### Requirement: 爆仓价计算
系统 SHALL 根据杠杆和入场价计算做多的爆仓价。

公式：爆仓价 = 入场价 × (1 - 1/杠杆)（全仓简化模型，不含资金费率）

#### Scenario: 杠杆为 5x
- **WHEN** 入场价 = 100 USDT，杠杆 = 5
- **THEN** 爆仓价 = 80.00 USDT

#### Scenario: 杠杆为 1x（无杠杆）
- **WHEN** 入场价 = 100 USDT，杠杆 = 1
- **THEN** 爆仓价 = 0（永不爆仓）

#### Scenario: 杠杆为 10x
- **WHEN** 入场价 = 50000 USDT，杠杆 = 10
- **THEN** 爆仓价 = 45000.00 USDT

### Requirement: 安全性评估
系统 SHALL 评估参数组合的安全性，返回三级结论：

- **安全**：网格下限 ≥ 爆仓价 × 1.1（下限在爆仓价下方有 10% 以上缓冲）
- **警告**：爆仓价 < 网格下限 < 爆仓价 × 1.1（缓冲不足 10%）
- **危险**：网格下限 ≤ 爆仓价（在设定区间内就会爆仓）

#### Scenario: 安全参数
- **WHEN** 爆仓价 = 80，网格下限 = 95
- **THEN** 结论为"安全"，显示缓冲比例

#### Scenario: 警告参数
- **WHEN** 爆仓价 = 80，网格下限 = 84
- **THEN** 结论为"警告"，提示缓冲不足 10%

#### Scenario: 危险参数
- **WHEN** 爆仓价 = 80，网格下限 = 75
- **THEN** 结论为"危险"，明确告知将爆仓

### Requirement: 网格分布测算
系统 SHALL 根据单次购买数量计算给定上下限内的网格分布。

- 网格数量 = ceil((上限 - 下限) / 网格大小)
- 单次网格名义价值 = 单次购买数量 × 入场价
- 单次网格利润 = 单次购买数量 × 网格大小（价差 × 数量，与杠杆无关）
- 总占用资金 = 单次网格名义价值 × 网格数量 / 杠杆（杠杆越高，实际占用保证金越少）

#### Scenario: 简单网格计算
- **WHEN** 上限 = 110，下限 = 90，网格大小 = 2，入场价 = 100，杠杆 = 5，单次购买 = 1 币
- **THEN** 网格数量 = 10，单次名义价值 = 100 USDT，单次利润 = 2 USDT，总占用 = 200 USDT

#### Scenario: 杠杆=1（现货）
- **WHEN** 上限 = 110，下限 = 90，网格大小 = 2，入场价 = 100，杠杆 = 1，单次购买 = 1 币
- **THEN** 网格数量 = 10，单次名义价值 = 100 USDT，单次利润 = 2 USDT，总占用 = 1000 USDT

#### Scenario: BTC 网格
- **WHEN** 上限 = 71500，下限 = 58500，网格大小 = 500，入场价 = 65000，杠杆 = 1，单次购买 = 0.01 币
- **THEN** 网格数量 = 26，单次名义价值 = 650 USDT，单次利润 = 5 USDT，总占用 = 16900 USDT

### Requirement: 初始资金校验
系统 SHALL 校验初始资金是否足够覆盖网格总占用资金，并在资金不足时给出提示。

- 剩余资金 = 初始资金 - 总占用资金
- 资金充足标志：初始资金 ≥ 总占用资金

#### Scenario: 资金充足
- **WHEN** 初始资金 = 20000，总占用 = 16900
- **THEN** 显示剩余 3100 USDT，无警告

#### Scenario: 资金不足
- **WHEN** 初始资金 = 10000，总占用 = 16900
- **THEN** 显示红色不足警告，提示超出 6900 USDT

### Requirement: 最大回撤估算
系统 SHALL 估算从入场价跌到网格下限的最大回撤。

公式：最大回撤 = (入场价 - 下限) / 入场价 × 杠杆 × 100%

#### Scenario: 下跌 20% 触发下限
- **WHEN** 入场价 = 100，下限 = 80，杠杆 = 5
- **THEN** 最大回撤 = 100%（即全仓亏损）

### Requirement: 结果展示与建议
系统 SHALL 在结果页清晰展示：初始资金、爆仓价、安全边际、安全等级、单次购买量、单次名义价值、单次利润、网格数量、总占用资金、剩余资金、最大回撤，以及参数调整建议。

#### Scenario: 安全结果展示
- **WHEN** 参数安全且资金充足
- **THEN** 结果页显示所有指标，建议区域为绿色提示"参数安全"，剩余资金绿色

#### Scenario: 危险结果展示
- **WHEN** 参数危险或资金不足
- **THEN** 结果页突出红色警告，建议区域包含具体可执行建议（如"建议降低杠杆至 3x"或"建议将下限提高至 X"），资金不足时顶部提示超出金额

### Requirement: 精度与单位
所有价格输出 SHALL 保留 2 位小数，金额使用 USDT 单位。系统内部计算 SHALL 使用 `decimal.Decimal` 保证精度。

#### Scenario: 小数精度
- **WHEN** 计算结果为 80.123456
- **THEN** 页面展示为 80.12 USDT
