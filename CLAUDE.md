# GridTrading — 加密货币网格交易测算工具

## 项目定位

为加密货币网格交易提供参数测算，帮助交易者在给定资金、杠杆、网格大小、上下限等条件下评估策略表现，核心目标是**避免爆仓**。

## 核心概念

- **网格交易（Grid Trading）**：在设定的价格区间内，按固定间距挂多空买卖单，机械地低买高卖，赚取波动的钱
- **杠杆（Leverage）**：借入资金放大仓位倍数，同时放大盈亏和爆仓风险
- **爆仓价（Liquidation Price）**：当亏损达到保证金被强制平仓的价格，这是测算工具的核心警戒线
- **保证金（Margin）**：用于维持仓位的抵押资金
- **网格大小（Grid Size）**：相邻两个网格挂单之间的价格间距
- **上下限（Upper/Lower Price）**：网格运行的价格区间

## 测算目标

用户输入一组参数，工具输出：
- 在当前参数下是否会爆仓（极端行情压力测试）
- 所需保证金是否充足
- 每个网格的收益 / 整体预期收益率
- 最大回撤估算
- 建议的安全参数范围（如最大安全杠杆）

## 技术栈

- **后端**：Python 3.x + Flask
- **模板**：Jinja2（Flask 内置）
- **前端**：原生 HTML + CSS（不使用 JS 框架，保持轻量）
- **部署**：`flask run` 本地开发即可，无需 Docker / 云服务
- **监听地址**：启动 Flask 测试时监听 `0.0.0.0`（非 `127.0.0.1`），便于局域网内其他设备访问测试
- **环境**：项目根目录建立 `venv` 虚拟环境，所有依赖安装在其中，`venv/` 加入 `.gitignore`

## 技术约定

- 货币单位：默认 USDT（稳定币计价），价格精度保留 2 位小数
- 方向：可选 做多，做空 ，中性网格
- 计算精度：使用 `decimal.Decimal` 高精度运算，避免浮点累积误差
- 语言：代码注释和文档默认中文，技术术语保留英文
- 数据库：回测功能使用 SQLite（`data/klines.db`，`.gitignore` 入库）缓存币安K线数据，无外部数据库依赖。测算功能仍纯内存运算，无持久化。
